# Content Writing — Zolai Platform

## Brand voice

- **Warm and scholarly** — like a knowledgeable community elder, not a corporate product
- **Inclusive** — written for Zomi diaspora, learners, researchers, and AI/ML contributors
- **Bilingual-aware** — English content should feel natural alongside Tedim Zolai text
- **Respectful** — the language and culture are sacred; never trivialize

## Tone by context

| Context | Tone |
|---|---|
| Landing / marketing | Inspiring, mission-driven, accessible |
| Tutor / learning | Encouraging, patient, Socratic |
| Wiki / grammar | Precise, academic, well-cited |
| Error messages | Clear, non-blaming, actionable |
| Admin / dashboard | Neutral, efficient, direct |

## UI copy rules

- Buttons: verb-first — "Search Dictionary", "Start Lesson", "Submit Entry"
- Empty states: explain what goes here + a call to action
- Error messages: say what happened + what to do — never "An error occurred"
- Loading states: "Loading dictionary…" not just "Loading…"
- Placeholders: show an example — `e.g. lungdam` not just "Search…"

## Wiki entry structure

```markdown
# [Term]

**Category:** grammar | phonology | morphology | dialect | culture

## Definition
One clear paragraph.

## Examples
- Zolai: [example sentence]
- English: [translation]

## Notes
Any dialect variation, historical context, or cross-references.
```

## SEO / meta copy

- Page titles: `[Feature] — Zolai AI | Tedim Zolai Language Platform`
- Meta descriptions: 150–160 chars, include "Tedim Zolai" and primary keyword
- OG titles: shorter, punchy — max 60 chars

## Bilingual content rules

- Zolai text always appears before English translation
- Use `—` (em dash) to separate: `Lungdam — Happy`
- Never mix dialects in the same content block
- Minimum font size 16px for Zolai body text (readability)
