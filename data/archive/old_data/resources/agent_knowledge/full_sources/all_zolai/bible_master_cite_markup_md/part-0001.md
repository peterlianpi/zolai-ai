# Manual for Data Row Expansion

## Markup Structure

Each row begins with a `keyword`. Rows may optionally include attributes and/or a description, separated from the keyword by an equal sign `=`. Rows are separated by a line break `\n`.

### Key Rules

1. Mandatory Equal Sign for Additional Data:
   * The keyword must be separated from attributes or a description by an equal sign =.
   * If the equal sign is missing, the row is treated as only a keyword, with no attributes or description.

2. Attributes:
   * Attributes, if present, must follow the equal sign and are formatted in parentheses (attribute:value).
   * Attributes are optional and can describe additional details about the keyword.

3. Description:
   * A description can be provided with or without attributes.
   * The description always comes after the equal sign =.

## Attributes

Attributes enhance the data row and provide additional context. Supported attributes include:

1. (t:type)
   * Specifies the type or category of the keyword (e.g., verb, noun, adjective).

2. (w:term)
   * The main term or word associated with the keyword.

3. (e:example)
   * An example sentence or context showing how the keyword is used.

4. (s:synonym)
   * A synonym for the keyword.

5. (a:antonym)
   * An antonym for the keyword.

6. (d:definition)
   * A detailed definition of the keyword.

7. (o:origin)
   * Indicates the origin or etymology of the keyword.

## Examples

Keyword Only:

```tsv
keyword
```

> Explanation:
This row contains only a keyword with no attributes or description.

Row With Description Only:

```tsv
keyword = This is a description of the keyword.
```

> Explanation:
The equal sign separates the keyword and its description.

Row With Attributes:

```tsv
keyword = (t:verb) (w:action) (s:perform) (a:cease)
```

> Explanation:
Attributes are provided after the equal sign, but no description is included.

Fully Expanded Row:

```tsv
keyword = (t:verb) (w:generate) (s:create) (e:He generated a new idea.) (a:destroy) (d:To bring into existence or produce.) (o:Latin, generare) Used to describe the act of producing or creating something.
```

> Explanation:
The row includes both attributes and a description, separated from the keyword by the equal sign.

### Key Points

1. Equal Sign Determines Row Structure:
   * Rows without an equal sign are treated as keyword-only entries.
   * Rows with an equal sign contain additional data (attributes, description, or both).

1. Consistency and Flexibility:
   * Attributes are optional but must follow the formatting rules if included.
   * Descriptions can be added with or without attributes.

1. Readability:
   * The consistent use of = ensures clear separation between the keyword and its associated data.
